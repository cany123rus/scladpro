# ScladPro Workflow Rules

Обязательные правила для всех новых задач по проекту ScladPro.

## 1) Backup перед любой новой задачей

Перед началом любых изменений обязательно:

1. Создать backup проекта в формате:
   `source_code_2_YYYY-MM-DD_HH-mm-ss`
2. Внутри backup создать файл `TASK.txt` с текстом задания пользователя.
3. Добавить запись в журнал версий: `backups/VERSIONS_LOG.md`:
   - дата/время
   - имя backup
   - текст задания
   - что было изменено
   - результат проверки (локально)
   - статус деплоя

## 2) Не деплоить сразу

Перед деплоем на хостинг обязательно:

1. Запустить проект локально на устройстве.
2. Прогнать проверку (минимум):
   - сборка (`npm run build`)
   - ключевой сценарий, связанный с задачей
3. Только после успешной локальной проверки выполнять деплой.

## Быстрые команды

Создать backup с задачей:

```bash
zsh /Users/alex/projects/source_code/scripts/scladpro_backup.sh "<текст задания>"
```

Добавить запись в журнал:

```bash
zsh /Users/alex/projects/source_code/scripts/scladpro_log_version.sh \
  "<имя backup>" "<что сделано>" "<проверка>" "<деплой статус>"
```
